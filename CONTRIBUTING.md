# Contributing to Arcade Metal Files

Thank you for your interest in contributing to this project! This document provides guidelines for contributing.

## How to Contribute

### Reporting Issues

- Use the [Issues](https://github.com/p0rgy8190/ArcadeMetal/issues) tab to report problems
- Include clear descriptions of the issue
- Attach reference images or specifications if applicable
- For file accuracy issues, include measurements and source documentation

### Submitting New Files

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b add/arcade-name-part-description
   ```
3. **Add your files** to the appropriate directory
4. **Include documentation** (README or specifications)
5. **Submit a Pull Request** with a clear description

## File Guidelines

### CAD Files

- Include multiple formats (DXF, DWG, STEP, PDF) when possible
- Clearly label all dimensions
- Use appropriate units (mm or inches - specify)
- Include scale information
- Add part numbers if available

### Documentation

- Include assembly instructions if applicable
- Reference original arcade specifications
- Note material specifications
- Include machine settings (tool paths, feeds/speeds if applicable)
- Add safety warnings if needed

### Naming Conventions

```
[ArcadeName]_[PartName]_[Version].[extension]
Example: Galaga_SidePanel_v1.dxf
```

## Code of Conduct

- Be respectful and collaborative
- Verify file accuracy before contributing
- Provide proper attribution for any referenced designs
- Follow licensing requirements

## Questions?

Open an issue or discussion for questions about contributing.
